cConvertBodyTo

Propriétés du cConvertBodyTo

Famille de composant

Transformation

Fonction

Le composant cConvertBodyTo convertit le corps du message en un type d'une classe donnée.

Objectif

Le cConvertBodyTo est utilisé pour convertir le corps d'un message en un type d'une classe donnée.

Basic settings

Target Class Name

Saisissez le nom du type de classe dans lequel vous souhaitez convertir le corps du message.

Utilisation

Le cConvertBodyTo est utilisé comme composant intermédiaire dans une Route.

Limitation

 

Scénario : Convertir le corps d'un fichier XML en org.w3c.dom.Document.class

Dans ce scénario, un composant cConvertBodyTo est utilisé pour convertir le corps d'un fichier XML en un document org.w3c.dom.Document.class. Un cBean importe le document org.w3c.dom.Document.class, vérifie son contenu et affiche le nom de l'élément racine ainsi que le contenu de chaque élément category.

Le fichier XML se présente ainsi :

<bookstore>
    <bookshelf>
        <category>Cooking</category>
        <quantity>100</quantity>
    </bookshelf>
    <bookshelf>
        <category>Languages</category>
        <quantity>200</quantity>
    </bookshelf>
    <bookshelf>
        <category>Arts</category>
        <quantity>300</quantity>
    </bookshelf>
    <bookshelf>
        <category>Science</category>
        <quantity>400</quantity>
    </bookshelf>
</bookstore>

Déposer les composants et les relier

  1. Glissez-déposez les composants suivants de la Palette dans l'espace de modélisation graphique : cFile, cConvertBodyTo et cBean.

  2. Reliez le cFile au cConvertBodyTo à l'aide d'un lien Row > Route.

  3. Reliez le cConvertBodyTo au cBean à l'aide d'un lien Row > Route.

  4. Nommez les composants comme vous le souhaitez afin de mieux identifier leur fonctionnalité.

Configurer les composants

  1. Double-cliquez sur le composant cFile nommé Read_message pour ouvrir sa vue Basic settings dans l'onglet Component.

  2. Dans le champ Path, saisissez le chemin d'accès ou parcourez votre système jusqu'au fichier XML source.

  3. Si le dossier du fichier source contient plus d'un fichier, saisissez le nom du fichier XML qui vous intéresse dans le champ FileName et laissez les autres paramètres tels qu'ils sont.

  4. Double-cliquez sur le composant cConvertBodyTo nommé Convert_message_body pour ouvrir sa vue Basic settings dans l'onglet Component.

  5. Dans le champ Target Class Name, saisissez le nom de votre classe cible, org.w3c.dom.Document.class dans ce scénario.

  6. Double-cliquez sur le composant cBean nommé Print_content, pour ouvrir sa vue Basic settings.

  7. Dans le champ Bean class, saisissez le nom du bean à invoquer, beans.PrintConvertToBean.class dans ce scénario.

    Notez que ce bean a déjà été défini dans le nœud Code du Repository et se présente comme suit :

    package beans;

import org.w3c.dom.Document;
import org.w3c.dom.Element;
import org.w3c.dom.NodeList;
public class PrintConvertToBean {

	/**
	 * print input message
	 * @param message
	 */
	public static void helloExample(Document message) {
		if (message == null) {
			System.out.println("There's no message here!");
			return;
		}
		Element rootElement = message.getDocumentElement();
		if (rootElement == null) {
			System.out.println("There's no root element here!");
			return;
		}
		System.out.println("The root element name is:"
				+ rootElement.getNodeName());
		System.out.println("The book categories are:");
		NodeList types = rootElement.getElementsByTagName("category");
		for(int i = 0;i<types.getLength();i++){
			Element child = (Element) types.item(i);
			System.out.println(child.getFirstChild().getNodeValue());
		}
	}
}

    Pour plus d'informations concernant la création et l'utilisation de Beans Java, consultez le Guide Utilisateur de Talend Open Studio for ESB/>.

  8. Appuyez sur les touches Ctrl+S pour sauvegarder votre Route.

Visualiser du code et exécuter la Route

  1. Cliquez sur l'onglet Code au bas de l'espace de modélisation graphique pour vérifier le code généré.

    Comme dans la capture d'écran, le corps du message de (from) l'endpoint Read_message est converti en org.w3c.dom.Document.class by cConvertBodyTo_1. Puis le org.w3c.dom.Document.class est traité par le .bean(beans.PrintConvertToBean.class) invoqué par cBean_1.

  2. Appuyez sur F6 pour exécuter la Route.

    Résultat : Le nom de l'élément racine ainsi que le contenu des éléments category sont affichés.
